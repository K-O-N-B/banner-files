### Step 3: 修改并提交

1. **修改版本号**：`"version": "v2.0.0"` → `"v2.0.1"`
2. **修改消息内容**：改成你想显示的文字
3. **修改图标**：选择合适的 Emoji（见下方图标表）
4. **点击 Commit changes** → 确认提交

## ⚙️ 常用参数

### 轮播速度

```json
"carousel": {
  "interval": 3000  // 3秒（快速）
  "interval": 5000  // 5秒（推荐）⭐
  "interval": 8000  // 8秒（慢速）
}
```

### 其他配置

```json
"carousel": {
  "enabled": true,        // 是否启用轮播
  "interval": 5000,       // 轮播间隔（毫秒）
  "pauseOnHover": true,   // 鼠标悬停时暂停
  "showIndicators": true, // 显示小圆点
  "loop": true            // 循环播放
}
```

## ❓ 常见问题

### Q: 如何修改轮播速度？

**A**: 修改 `carousel.interval` 参数

```json
"carousel": {
  "interval": 3000  // 3秒
}
```

---

### Q: 如何关闭轮播，使用单条模式？

**A**: 使用单条模板（模板 C）或设置 `carousel.enabled: false`

---

### Q: 版本号什么时候需要更新？

**A**: **每次修改消息内容时都要更新版本号**

```json
// 修改前
"version": "v2.0.0"

// 修改后
"version": "v2.0.1"  // ← 版本号+1
```

**原因**: 只有版本号变化，用户才会看到 NEW 标签

---

### Q: 我的轮播没有显示？

**A**: 检查这4项：

1. ✅ `enabled: true`
2. ✅ `carousel.enabled: true`
3. ✅ `messages` 数组至少有 1 条消息
4. ✅ JSON 格式正确（用 https://jsonlint.com/ 验证）

## 🎯 最佳实践

### ✅ 推荐

- 消息数量：**2-3 条**（最佳体验）
- 轮播间隔：**5 秒**（标准配置）
- 消息长度：**20-40 字**（易读）
- 使用标签：**【重要】、【提示】**（分类清晰）

### ❌ 避免

- ❌ 消息过多（>5条）
- ❌ 轮播过快（<2秒）
- ❌ 消息过长（>80字）
- ❌ 不更新版本号
