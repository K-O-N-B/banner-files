## 🚀 快速开始

### 第一步：打开 GitHub 仓库

1. 访问你的配置仓库：`https://github.com/K-O-N-B/banner-files`
2. 找到 `banner-config.json` 文件
3. 点击 **编辑按钮** ✏️（右上角）

### 第二步：选择配置模式

你有两种配置模式可选：

#### 模式 A：单条消息模式（原有模式）

```json
{
  "version": "v1.0.0",
  "icon": "⚠️",
  "message": "这是一条单条消息",
  "enabled": true
}
```

#### 模式 B：轮播模式（新功能）⭐

```json
{
  "version": "v2.0.0",
  "enabled": true,
  
  "carousel": {
    "enabled": true,
    "interval": 5000
  },
  
  "messages": [
    {
      "id": "msg-1",
      "icon": "⚠️",
      "message": "第一条消息"
    },
    {
      "id": "msg-2",
      "icon": "🎉",
      "message": "第二条消息"
    }
  ]
}
```

### 第三步：保存并提交

1. 点击 **Commit changes**
2. 填写提交说明（可选）
3. 点击 **Commit changes** 确认

### 第四步：等待同步

- ⏰ 所有用户会在 **30秒内** 自动同步你的更新
- 🌐 Banner 会显示 **🌐 在线** 标签
- 🆕 新版本会显示 **NEW** 标签（3秒后自动消失）

## 📋 配置文件格式

### 完整配置结构

```json
{
  // ========== 基础配置 ==========
  "version": "v2.0.0",           // 版本号（必填）
  "enabled": true,               // 是否启用（必填）
  "updated_at": "2025-10-11T12:00:00Z",  // 更新时间（可选）
  "description": "配置说明",      // 描述（可选）
  
  // ========== 轮播配置 ==========
  "carousel": {
    "enabled": true,             // 是否启用轮播（必填）
    "interval": 5000,            // 轮播间隔（毫秒）
    "pauseOnHover": true,        // 鼠标悬停时暂停
    "showIndicators": true,      // 显示指示器（小圆点）
    "loop": true                 // 循环播放
  },
  
  // ========== 消息列表 ==========
  "messages": [
    {
      "id": "msg-1",             // 消息唯一标识（必填）
      "icon": "⚠️",              // 图标 Emoji（必填）
      "message": "消息内容"       // 消息文本（必填）
    }
  ]
}
```

## 🔧 参数详细说明

### 1️⃣ 基础配置参数

#### `version` （版本号）

- **类型**: `string`
- **必填**: ✅ 是
- **格式**: `v主版本号.次版本号.修订号` 或 `主版本号.次版本号.修订号`
- **作用**: 控制 Banner 的强制更新

**重要说明**：
- ✅ 支持格式：`v1.0.0` 或 `1.0.0`（系统会自动规范化）
- ✅ 当版本号变化时，会自动清除用户的关闭记录，**强制显示** Banner
- ✅ 即使用户之前点击了 ❌ 关闭，新版本也会重新显示

**示例**：
```json
"version": "v1.0.0"  // ✅ 推荐格式
"version": "1.0.0"   // ✅ 也支持
"version": "v2.1"    // ✅ 支持两位版本号
```

**版本更新策略**：
- 重要提示更新：`v1.0.0` → `v2.0.0`
- 小幅度更新：`v1.0.0` → `v1.1.0`
- 修正文字：`v1.0.0` → `v1.0.1`

---

#### `enabled` （是否启用）

- **类型**: `boolean`
- **必填**: ✅ 是
- **默认值**: `true`

**作用**：
- `true`: Banner 正常显示
- `false`: Banner 完全隐藏（所有用户）

**使用场景**：
```json
// 场景 1: 正常启用
{
  "version": "v1.0.0",
  "enabled": true
}

// 场景 2: 临时关闭所有 Banner
{
  "version": "v1.0.0",
  "enabled": false  // 所有用户的 Banner 会立即隐藏
}
```

---

#### `updated_at` （更新时间）

- **类型**: `string`
- **必填**: ❌ 否（推荐填写）
- **格式**: ISO 8601 标准时间格式

**示例**：
```json
"updated_at": "2025-10-11T12:00:00Z"
"updated_at": "2025-10-11T12:30:00+08:00"  // 支持时区
```

**作用**：仅用于记录和管理，不影响显示逻辑

---

#### `description` （描述信息）

- **类型**: `string`
- **必填**: ❌ 否
- **作用**: 配置文件的说明性文字，方便管理员理解配置用途

**示例**：
```json
"description": "双十一活动Banner - 2条消息轮播"
```

---

### 2️⃣ 轮播配置参数 (`carousel`)

#### `carousel.enabled` （启用轮播）

- **类型**: `boolean`
- **必填**: ✅ 是（启用轮播模式时）
- **默认值**: `false`

**作用**：
- `true`: 启用轮播模式，使用 `messages` 数组
- `false`: 单条模式，使用旧的 `icon` 和 `message` 字段

**重要**：
```json
// ✅ 正确：启用轮播
{
  "carousel": {
    "enabled": true
  },
  "messages": [...]  // 必须提供消息数组
}

// ✅ 正确：单条模式（兼容旧配置）
{
  "icon": "⚠️",
  "message": "单条消息"
}

// ❌ 错误：启用轮播但没有消息
{
  "carousel": {
    "enabled": true
  }
  // 缺少 messages 数组
}
```

---

#### `carousel.interval` （轮播间隔）

- **类型**: `number`
- **必填**: ❌ 否
- **默认值**: `5000` (5秒)
- **单位**: 毫秒 (ms)
- **推荐范围**: `2000` - `20000` (2-20秒)

**作用**：控制轮播的自动切换速度

**常用值参考**：

| 间隔时间 | 毫秒值 | 适用场景 |
|---------|-------|----------|
| 2秒 | `2000` | 紧急提醒、短消息 |
| 3秒 | `3000` | 快速轮播、2-3条消息 |
| 5秒 | `5000` | **推荐值**，适合大多数场景 |
| 8秒 | `8000` | 较长消息、详细说明 |
| 10秒 | `10000` | 慢速播放、4+条消息 |

**示例**：
```json
"carousel": {
  "enabled": true,
  "interval": 5000  // 5秒切换一次
}
```

---

#### `carousel.pauseOnHover` （鼠标悬停暂停）

- **类型**: `boolean`
- **必填**: ❌ 否
- **默认值**: `true`

**作用**：
- `true`: 鼠标悬停在 Banner 上时暂停轮播
- `false`: 忽略鼠标悬停，持续轮播

**用户体验**：
- ✅ `true`（推荐）: 用户可以仔细阅读当前消息
- ⚠️ `false`: 适合不需要用户交互的场景

**示例**：
```json
"carousel": {
  "enabled": true,
  "interval": 5000,
  "pauseOnHover": true  // 推荐开启
}
```

---

#### `carousel.showIndicators` （显示指示器）

- **类型**: `boolean`
- **必填**: ❌ 否
- **默认值**: `true`

**作用**：
- `true`: 显示轮播指示器（小圆点）
- `false`: 隐藏指示器

**视觉效果**：
```
启用指示器（true）:
┌─────────────────────────────────┐
│ ⚠️ 重要提示...    ● ○ ○  ‹ › ✕ │
└─────────────────────────────────┘

禁用指示器（false）:
┌─────────────────────────────────┐
│ ⚠️ 重要提示...         ‹ › ✕   │
└─────────────────────────────────┘
```

**建议**：
- ✅ 2-4条消息：建议 `true`
- ⚠️ 单条消息：自动隐藏（无需设置）
- ⚠️ 5+条消息：可选 `false` 节省空间

---

#### `carousel.loop` （循环播放）

- **类型**: `boolean`
- **必填**: ❌ 否
- **默认值**: `true`

**作用**：
- `true`: 循环播放（最后一条→第一条）
- `false`: 单次播放（播放到最后一条后停止）

**行为差异**：

```json
// 循环模式（loop: true）
消息 1 → 消息 2 → 消息 3 → 消息 1 → ...（持续循环）

// 单次模式（loop: false）
消息 1 → 消息 2 → 消息 3 → 停止
```

**使用场景**：
- ✅ `true`: 适合常驻提示、重复播放
- ✅ `false`: 适合欢迎引导、一次性提示

**示例**：
```json
// 循环播放（推荐）
"carousel": {
  "enabled": true,
  "loop": true
}

// 单次播放（欢迎引导）
"carousel": {
  "enabled": true,
  "loop": false  // 播放3条消息后停止
}
```

### 3️⃣ 消息列表参数 (`messages`)

#### `messages` 数组

- **类型**: `array`
- **必填**: ✅ 是（轮播模式时）
- **最少**: 1 条消息
- **推荐**: 2-4 条消息
- **最多**: 不限（但建议不超过 5 条）

**每条消息包含**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 消息唯一标识 |
| `icon` | string | ✅ | Emoji 图标 |
| `message` | string | ✅ | 消息文本 |

---

#### `messages[].id` （消息标识）

- **类型**: `string`
- **必填**: ✅ 是
- **格式**: 建议使用 `msg-1`, `msg-2`, `tip-1` 等

**作用**：
- 唯一标识每条消息
- 便于配置管理和调试

**命名建议**：
```json
"messages": [
  { "id": "msg-1", ... },    // 通用消息
  { "id": "msg-2", ... },
  { "id": "alert-1", ... },  // 警告类消息
  { "id": "tip-1", ... },    // 提示类消息
  { "id": "news-1", ... }    // 新闻类消息
]
```

**注意**：
- ⚠️ 每个 `id` 必须唯一
- ⚠️ 不要使用特殊字符（建议只用字母、数字、连字符）

---

#### `messages[].icon` （消息图标）

- **类型**: `string`
- **必填**: ✅ 是
- **格式**: Emoji 表情符号

**常用图标参考**：

| 类型 | Emoji | 含义 |
|------|-------|------|
| 警告 | ⚠️ | 重要提示 |
| 禁止 | 🚫 | 禁止操作 |
| 成功 | ✅ | 完成/成功 |
| 信息 | ℹ️ | 一般信息 |
| 庆祝 | 🎉 | 新功能/活动 |
| 灯泡 | 💡 | 提示/技巧 |
| 火箭 | 🚀 | 更新/升级 |
| 喇叭 | 📢 | 通知/公告 |
| 星星 | ⭐ | 亮点/推荐 |
| 文档 | 📖 | 文档/教程 |
| 工具 | 🔧 | 维护/修复 |
| 火焰 | 🔥 | 热门/紧急 |

**示例**：
```json
"messages": [
  {
    "id": "warning",
    "icon": "⚠️",
    "message": "重要警告"
  },
  {
    "id": "celebrate",
    "icon": "🎉",
    "message": "新功能发布！"
  }
]
```

**获取更多 Emoji**：
- 🌐 访问：https://emojipedia.org/
- 🔍 搜索你需要的表情
- 📋 直接复制粘贴到配置文件

---

#### `messages[].message` （消息文本）

- **类型**: `string`
- **必填**: ✅ 是
- **推荐长度**: 20-80 字符
- **最大长度**: 无限制（但会影响显示效果）

**编写建议**：

1. **简洁明了**：
   ```json
   // ✅ 好
   "message": "【重要】不要使用官方 tab 保存功能！"
   
   // ❌ 不好（太长）
   "message": "特别重要的提示：在使用本插件的过程中，请务必注意不要使用官方提供的tab功能进行保存操作，否则可能会导致数据错误..."
   ```

2. **使用标签分类**：
   ```json
   "message": "【重要】主要内容"
   "message": "【提示】提示内容"
   "message": "【新功能】功能说明"
   "message": "【更新】更新说明"
   ```

3. **突出关键词**：
   ```json
   "message": "⚡ 紧急：服务器维护中"
   "message": "🎉 新增：多标签管理"
   "message": "💡 技巧：使用快捷键 Ctrl+S"
   ```

4. **避免使用特殊字符**：
   ```json
   // ✅ 好
   "message": "使用引号说「重要」"
   
   // ⚠️ 注意转义
   "message": "使用引号说\"重要\""  // JSON 需要转义
   ```

## ❓ 常见问题解答

### Q1: 如何更改轮播速度？

**A**: 修改 `carousel.interval` 参数（单位：毫秒）

```json
"carousel": {
  "interval": 3000  // 3秒（快速）
  "interval": 5000  // 5秒（推荐）
  "interval": 8000  // 8秒（慢速）
}
```

---

### Q2: 如何临时关闭轮播，回到单条模式？

**A**: 有两种方法：

**方法 1**：设置 `carousel.enabled = false`
```json
{
  "version": "v1.0.0",
  "carousel": {
    "enabled": false
  },
  "icon": "⚠️",
  "message": "单条消息"
}
```

**方法 2**：直接使用旧格式（自动识别）
```json
{
  "version": "v1.0.0",

### Q3: 版本号什么时候需要更新？

**A**: 以下情况需要更新版本号：

| 情况 | 是否更新版本号 | 原因 |
|------|---------------|------|
| **修改消息内容** | ✅ 需要 | 让用户看到最新内容 |
| **添加/删除消息** | ✅ 需要 | 内容变化，需要强制刷新 |
| **修改轮播速度** | ❌ 不需要 | 仅改变行为，不影响内容 |
| **修改暂停设置** | ❌ 不需要 | 仅改变交互，不影响内容 |
| **重要紧急通知** | ✅ 必须 | 强制所有用户看到 |

**版本号递增规则**：
```
v1.0.0 → v1.0.1  // 小改动（修正文字）
v1.0.0 → v1.1.0  // 中等改动（添加消息）
v1.0.0 → v2.0.0  // 大改动（重大更新）
```

- 🌐 Banner 会显示 **🌐 在线** 标签
- 🆕 新版本会显示 **NEW** 标签（3秒后消失）

```

### Q6: 如何让 Banner 只显示一次（不循环）？

**A**: 设置 `carousel.loop = false`

```json
"carousel": {
  "enabled": true,
  "interval": 5000,
  "loop": false  // 播放完所有消息后停止
}
```

**效果**：
```
消息 1 (5秒) → 消息 2 (5秒) → 消息 3 (5秒) → 停止
```

## 🔧 故障排查

### 问题 1：JSON 格式错误

**症状**：修改后 Banner 没有更新

**原因**：JSON 格式不正确

**解决方法**：

1. 使用在线工具验证：https://jsonlint.com/
2. 检查常见错误：
   ```json
   // ❌ 错误：最后一项后面有逗号
   "messages": [
     { "id": "msg-1", ... },
     { "id": "msg-2", ... },  // ← 多余的逗号
   ]
   
   // ✅ 正确：最后一项没有逗号
   "messages": [
     { "id": "msg-1", ... },
     { "id": "msg-2", ... }
   ]
   ```

---

### 问题 2：版本号没有触发更新

**症状**：修改了配置但用户没有看到 NEW 标签

**原因**：版本号没有变化

**解决方法**：

确保版本号递增：
```json
// 旧配置
"version": "v1.0.0"

// 新配置（修改后）
"version": "v1.0.1"  // ✅ 版本号增加
```

### 问题 5：离线模式（显示 📦 离线）

**症状**：Banner 显示 `📦 离线` 而不是 `🌐 在线`

**原因**：
- GitHub 请求失败
- 网络不稳定
- GitHub 仓库无法访问

**解决方法**：
1. 检查 GitHub 仓库是否公开
2. 检查配置文件 URL 是否正确
3. 等待网络恢复（系统会使用缓存数据）

**缓存说明**：
- ✅ 系统会自动缓存最后一次成功的配置
- ✅ 离线时使用缓存数据
- ✅ 网络恢复后自动切换回在线模式

## 📚 最佳实践总结

### ✅ 推荐做法

1. **版本号管理**
   - 每次修改内容都更新版本号
   - 使用语义化版本号（v1.0.0）

2. **消息数量**
   - 保持 2-3 条消息（最佳）
   - 最多不超过 5 条

3. **轮播速度**
   - 短消息：3 秒
   - 中等消息：5 秒（推荐）
   - 长消息：8 秒

4. **用户体验**
   - 启用鼠标悬停暂停
   - 显示轮播指示器
   - 循环播放

5. **内容编写**
   - 简洁明了
   - 使用标签分类
   - 突出关键词

### ❌ 避免做法

1. ❌ 消息过长（>80字）
2. ❌ 轮播过快（<2秒）
3. ❌ 消息过多（>5条）
4. ❌ 频繁更新版本号（让用户厌烦）
5. ❌ 不更新版本号（用户看不到新内容）
